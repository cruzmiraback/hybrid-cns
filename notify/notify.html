BridgeIt Contextual Notification System

Building on basic push capabilities, which are commonplace in modern web and mobile applications, contextual notification vaults you beyond the basics, enabling you to tailor notifications precisely to the individual circumstances of your users. The BridgeIt Contextual Notification System offers a suite of cloud-based services for building sophisticated, context-sensitive notification capabilities into your HTML5 applications. With the Contextual Notification System, you can:

  • Customize a secure notification realm to match your application's exact requirements,
  • Leverage realm services to establish and maintain dynamic application and user context,
  • Generate and filter notifications against that context,
  • Push notifications to users over a wide range of transports, including Email, SMS, Ajax Push, and Cloud Push (using mobile platform push services including APNS, GCM, WNS, and BBPS),
  • Configure, manage, and monitor your notification realms from the BridgeIt Console.

Cloud-based hosting eliminates deployment hassles, making it easy for you to add contextual notification to your HTML5 applications today, and ensuring that your users are getting timely notifications, tailored to their specific circumstance. Let BridgeIt help you get the right information to the right people at the right time.

Secure Notification Realms

The Contextual Notification System is founded on a multi-tenant, realm-based security architecture. Administrative accounts are established for each tenant, with namespace and database separation enforced between accounts. Within an account, multiple notification realms can be established, again with namespace and data separation enforced between realms. Within a notification realm you can configure the required services, add and manage users, establish and maintain user and application context, and generate notifications using customized logic to filter against that context. The security features of a notification realm include:

  • Complete sandboxing and namespace separation between realms,
  • User authentication with support for Basic, Digest, OAuth2 protocols, and integration with 3rd party authentication services like Facebook, Twitter, and Google,
  • Permission-based access control using time-limited access tokens,
  • CORS support with configurable, realm-level origin checking.

Realm Services

The Contextual Notification System offers a suite of cloud-based services for the implementation of your business-specific notification logic. Each service presents a REST API which is conveniently accessible from your HTML5 application via a JavaScript library. The following Realm Services are available:

Authentication Service manages user credentials, authenticates users, and enforces permission-based access control to other Realm Services.

Document Service provides CRUD operations for JSON documents and document query capabilities. Application and user context can be persisted in the Document Service, and subsequent queries against that context can be used to filter notifications.

Locate Service manages location-related context, including user location, regions and points of interest. Location monitors can be configured to fire events under specific conditions like location change, entering/exiting regions, or coming into proximity of a point of interest. These events can directly trigger notifications, or can initiate flows in the Code Service. The Location Service supports various location sources including BridgeIt Geotracking and Google Locate, and is extensible to other 3rd party location sources.

Code Service provides a flow-based programming environment for the development of custom server-side logic in your application. A suite of input, output and function nodes make it easy to develop flows to handle user interactions, process location data and monitors, filter against context, and generate notifications. A drag and drop flow editor facilitates development of custom flow logic without writing any code.

Notification Service pushes notifications over multiple transports, automatically selecting the appropriate transport for each client receiving a notification. Supported transports include Email, SMS, Ajax Push, and Cloud Push via platform specific push mechanisms including Apple Push Notification Service, Google Cloud Messaging, Windows Notification Service, and Blackberry Push Service.

Metrics Service collects metrics from other running services in support of run time analytics for notification realms and accounts. The Metrics Service can also be used to generate application-specific metrics.

BridgeIt Console

The Console provides administrative and operational control over accounts, and realms in the Contextual Notification System. Administrative capabilities include:

  • CRUD operations for notification realm,
  • CRUD operations for users,
  • Permissions and access control management.

Operational capabilities include:

  • Manual and scheduled notification generation,
  • CRUD operations for geolocation regions, points of interest, and monitors,
  • CRUD operations for context filters,
  • Code Service flow editing and management,
  • Metrics viewing and analytics.
